# Meeting Intelligence (Интеллектуальный анализ встреч)

## Описание
Meeting Intelligence — это современное веб-приложение, которое использует возможности искусственного интеллекта (Google Gemini API) для автоматического анализа аудио- и видеозаписей ваших рабочих встреч, звонков и совещаний. Система берет на себя рутину по составлению протоколов (meeting minutes), позволяя участникам сфокусироваться на живом обсуждении, а не на ведении заметок.

## Функциональность системы

*   **Загрузка записей:** Поддержка загрузки аудио- и видеофайлов напрямую через удобный веб-интерфейс.
*   **Умная транскрибация:** Автоматическое распознавание речи и перевод её в текст с разделением по спикерам (SPEAKER_1, SPEAKER_2 и т.д.) и фиксацией таймкодов.
*   **Генерация саммари (Summary):** Создание краткого содержания встречи, автоматическое выделение ключевых тем обсуждения и зафиксированных решений.
*   **Извлечение задач (Action Items):** ИИ автоматически находит в разговоре постановки задач, определяет ответственных лиц и сроки выполнения (дедлайны).
*   **Интерактивный ИИ-чат по встрече:** Возможность задавать вопросы ИИ-ассистенту строго по контексту прошедшей встречи. Например: *"Какие риски мы обсуждали?"*, *"Сделай выжимку для руководства"* или *"О чем договорились в конце?"*.
*   **База знаний:** Удобный дашборд с историей всех ранее загруженных и обработанных совещаний с возможностью быстрого поиска нужной информации.

## Юз кейсы (Сценарии использования)

### 1. Управление проектами (Project Management)
**Проблема:** После каждого статус-митинга или планирования спринта менеджеру приходится тратить 20-30 минут на написание фоллоу-апа (follow-up) и перенос задач в таск-трекер.
**Решение:** Менеджер загружает запись звонка в систему. Meeting Intelligence автоматически формирует список задач с назначенными исполнителями и сроками, а также выдает готовое саммари для отправки команде в мессенджер.

### 2. Сбор требований (Бизнес и Системный анализ)
**Проблема:** Во время интервью с заказчиком (Discovery phase) легко упустить важные детали, если пытаться одновременно вести активный диалог и делать подробные заметки.
**Решение:** Аналитик полностью погружается в беседу, записывая звонок. После загрузки файла он может использовать ИИ-чат для быстрого поиска ответов: *"Какие нефункциональные требования назвал клиент?"* или *"Какие ограничения интеграции обсуждались?"*.

### 3. HR и Рекрутинг
**Проблема:** При проведении множества собеседований информация забывается, сложно вспомнить детали ответов конкретного кандидата при финальном сравнении.
**Решение:** Загрузка записи технического или HR-интервью. Нанимающий менеджер может быстро просмотреть саммари или спросить в чате: *"Как кандидат ответил на вопрос про разрешение конфликтов в команде?"* или *"Какие зарплатные ожидания были озвучены?"*.

### 4. Отдел продаж (Sales & Account Management)
**Проблема:** Анализ звонков с клиентами для выявления их истинных болей и потребностей занимает слишком много времени.
**Решение:** Менеджер по продажам загружает запись демо-звонка и мгновенно получает список ключевых возражений клиента, а также четкие договоренности о следующих шагах (Next Steps).

## Технологический стек
*   **Frontend:** React 19, React Router, Tailwind CSS (для стилизации), Lucide React (иконки).
*   **Backend:** Node.js, Express, SQLite (для надежного локального хранения истории встреч и транскрипций).
*   **AI Engine:** Google Gemini API (модель `gemini-3-flash-preview`) для мультимодального анализа файлов, транскрибации, структурирования данных и работы контекстного чат-бота.
